# Reference Documentation

This directory contains authoritative reference documentation for core concepts, models, and principles in the Change-Driven Development system.

## Core Concepts

### [DECISIONS.md](DECISIONS.md)
The decision model defining the fundamental architecture:
- Tasks as units of work
- Change requests and versioning
- Approval workflow
- Quality gates

This is the **master contract** for the system.

### [SECURITY.md](SECURITY.md)
Security model and sandbox architecture:
- Path validation and allowlisting
- Command filtering
- Timeout enforcement
- Directory traversal prevention

Reference documents are:

- **Stable** - Rarely change, represent core design decisions
